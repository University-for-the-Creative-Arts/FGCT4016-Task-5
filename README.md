# FGCT4016 Task 5 'Data & Game Systems'

## 1. Introduction

The task outlined was to create an Unreal Engine 5 project that interacts with an online service tool by either sending an HTTP Request, using delegates, using json, using data tables, using CSV, or using data assets. At least one of those functions need to be done. The aforementioned functions and features are useful as they are used for online leaderboards, matchmaking and friend systems. Without these functions and features, most if not all multiplayer games would not be feasible, so it is important to know these sorts of features, especially if a future project may have online functionality.

## 2. Implementation

The task began with creating a Postgres database hosted on ***[Supabase](https://supabase.com/)***. A table was made within a schema in the project, and two variables were set in the table: `id (int8)` and `score (int8)`, with id set as the primary key as its a unique identifier as opposed to score, which you would expect duplicate entries if this were to be an actual game with scores.

`Row Level Security` was disabled for the table. This is to ensure data can be transferred without being blocked and fiddling with access keys for an unreasonable amount of time. But for an actual project, this may be bad practice for security reasons, but for the sake of the task, it was simpler to remove it than work around it.

A few rows were made in the table with different data for testing purposes

 ![Alt text](./gitimages/table.png "supabase table")

[ Figure 1. Supabase Table.]

Then, an actor is made which uses the `HTTP Request` node from the HTTP Request plugin. A URL that queries the database to get the score of the row with id = 1. 

To ensure it connects to the database, a `Make Request Header` node to input the API key is added to ensure the project connects with the supabase database.

 ![Alt text](./gitimages/testactor.png "Test Actor Blueprint")

[ Figure 2. Test Actor Blueprint.]

When ran and the actor is placed, it successfully outputs the score of id = 1, which is 5. If you were to change the query to `id=eq. 2`, it would output 10


## 3. Outcome

The outcome of the task is a program that queries a database hosted on ***[Supabase](https://supabase.com/)*** to get the score of a specified ID, and prints that score with a print string when the actor containing the logic is placed in the world. The logic used is expandable, to allow for posts or more complex get requests, which can be used for a scoreboard of an online game, or a game with high scores/fastest times.
